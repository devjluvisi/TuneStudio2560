The main header file for the application. Includes important variables and methods that are shared between all TuneStudio files.
Note that method definitions described in this file are declerations for main.cpp methods only.

◆ FAST_ADC

#define FAST_ADC   (PRGM_MODE == 2 ? true : false)

A macro for defining if fast analog read should be used.
Faster Analog Read takes up around 30-40 more bytes of flash but allows much faster analog read.
Increases the iterations per second (IPS) of creator mode by around 300-400.

Automatically enabled in PRGM_MODE == 2

◆ PERF_METRICS

#define PERF_METRICS   false

Enable/Disable performance metrics for TuneStudio2560.
Enabling this will: Enable serial console prints on the current Ram Usage, Stack Size, and Heap Fragmentation, as well as provide metrics on how fast certain functions are executing in micro and milliseconds as well as the change since last execute.

NOTE: Enabling performance metrics REQUIRES debug to be true.

◆ buttonFrequencies_t

typedef struct buttonFrequencies buttonFrequencies_t

Represents a structure for each different tune button in the program.

Each buttonFrequency has a "pin" describing the physical connection to the arduino, and an array of notes which stores all of the possible notes for that tune button.

◆ note_t

typedef struct note note_t

Represents a structure of a "note" in the program. (Used as note_t)

Notes are defined as having both a human readable (char string pointer) pitch and a 16-bit integer which represents the frequency of the note to be played by the passive buzzer.

uint16_t get_current_freq ( )

Get the current frequency from the potentiometer. Whatever tune button was pressed previously is not accounted for.

Remarks
This method does not deal with any tune buttons, rather it simply returns the current value as measured by the arduino.
This method's speed increases drastically when PRGM_MODE==2 or fast analog read is enabled.
Returns
The current frequency read by the potentiometer. (0 and 1023)

Set the selected page for selecting a song.

Each page selection represents a jump of 5.
EX: Page 1 (Songs 1-5)
Page 2 (Songs 6-10)

There can only be a absolute maximum of 51 pages because 255/5 is 51 and there are 255 maximum allowed songs.

◆ lcd

LiquidCrystal_I2C lcd
extern

Represents a global instance of the Liquid Crystal display used for TuneStudio2560.

The Liquid Crystal display uses the I2C protocol and is responsible for direct user feedback on using the program.
The display should be the recommended 20 columns and 4 rows.

Editing TuneStudio2560 to accomidate smaller displays is possible by adjusting the LCD_COLS and LCD_ROWS but some changes to setCursor(x, x) methods would need to be done.

Represents the global song object to be used in the program when managing songs.

This song object is used between program states and can be accessed at any time.
When a new song wants to be used by the application, the song is cleared using .clear() and notes are re-added.

Since
v1.2.0-R3 Songs are no longer pointers and are no longer dynamically allocated. Any song the program is going to use is now accessible by the "prgmSong" global variable.
