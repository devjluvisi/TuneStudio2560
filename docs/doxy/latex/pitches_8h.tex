\doxysubsection{Detailed Description}
A complete fill containing all of the human readable pitches available in Tune\+Studio2560. 

\begin{DoxyAuthor}{Author}
Jacob Lu\+Visi
\end{DoxyAuthor}
This header file was created to store constants for every char string (c-\/string) pitch which has a respective frequency.~\newline
 This is preferable for microcontrollers with low RAM due to the fact that the previous method\+:~\newline
 const button\+Frequencies\+\_\+t PROGRAM\+\_\+\+NOTES\mbox{[}TONE\+\_\+\+BUTTON\+\_\+\+AMOUNT\mbox{]} PROGMEM\{~\newline
 \{BTN\+\_\+\+TONE\+\_\+1, \{\{\char`\"{}\+B0\char`\"{}, 31\}, ...\}\},~\newline
 \};~\newline
 Stored the pointers for the pitches in PROGMEM but the pitch data was still stored in SRAM causing over 357 of SRAM bytes to be used. ~\newline
 This header file is dedicated to making sure the pitch data and the pitch pointers remain and are retrieved from PROGMEM. ~\newline
